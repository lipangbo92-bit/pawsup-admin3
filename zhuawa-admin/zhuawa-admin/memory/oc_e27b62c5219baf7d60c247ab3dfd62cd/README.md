# 群聊记忆 - 宠物店预约小程序项目

## 项目背景
- 项目类型：微信小程序 - 到店服务预约系统
- 技术栈：微信云开发 + 全款支付
- 核心功能：预约管理、技师排班、宠物管理、微信服务号通知

## 产品设计讨论（进行中）

### 1. 核心业务流程 - 已确认
- 预约路径：有技师=技师→服务→时间，无技师=服务→技师→时间
- 时间选择：30分钟为单位，已预约时间不可选
- 支付：全款支付
- 取消政策：提前3小时可取消全额退款，3小时内不可取消

### 2. 用户角色 - 已确认
- 顾客：微信一键登录
- 技师：小程序内置，查看自己预约、可操作取消
- 管理员：账号密码登录
- 小程序统一入口，通过账号区分权限

### 3. 数据模型 - 已确认（6张表）
- users, pets, services, technicians, schedules, appointments
- 排班按天，分早晚班，按小时

### 4. 关键交互 - 已确认
- 预约流程：有技师=技师→服务→时间，无技师=服务→技师→时间
- 支付：微信支付
- 服务号通知：预约成功/取消/改约 + 服务前提醒

### 5. 边界情况 - 已确认
- 预约时间冲突：显示不可选
- 取消政策：3小时前全额退款，3小时内不退款
- 技师请假/临时调整：管理员操作改约或取消
- 退款：原路返回微信支付
